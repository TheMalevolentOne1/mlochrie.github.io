The self guided lab aims to extend your knowledge of the subject by advancing and building on skills previously learnt. You are expected to integrate GitHub into your project workflow. Introduction

As previously introduced, GitHub is an important aspect of any Computer Science related subject. In the activities you will be shown how to integrate this process in one of two ways.

Activities

The following activities have been designed to extend your knowledge of version control using GitHub. Version control is an important aspect of any Computer Science related subject, where you work together on projects, develop complex systems, keep track of what is the truth (of files). In this module GitHub will be assessed in assignment 1.

In these activities you will be introduced to two methods of integrating GitHub into your workflows. You can obvious write GitHub commands using the Command Line (but this is out of scope of this module - you can however, do this if you have the knowledge of want to experiment). That said, you will be introduced to GitHub Desktop and integrating GitHub into Visual Studio Code.

Activity 1: Integrating GitHub Desktop into workflow

GitHub Desktop is a piece of software written by GitHub that allows users to use the GUI to initialise their projects and performs commits.

  • Download GitHub Desktop

  • Initialise GitHub by 'creating a new repository on your hard drive'

  • Provide a name for your repository and a short description. For example; 'Learning GitHub' and 'Learning how to integrate GitHub into my workflow for CO2509'

  • Specify where you want the directory to be created

  • Check 'Initialize this repository with a README'

  • Leave Git ignore selected to 'none' for now. You will revisit this later

  • Leave License set to 'none'

  • Click 'Create repository'

    This will now create the directory where all your files will be created and committed to GitHub

  • Navigate to that directory using Windows File Explorer, you should notice two files in the directory (REAME and .gitattributes)

  • Now publish the files to GitHub ('publish repository'), you will now need to authenticate your account credentials to perform the commit.

    • Sign into GitHub.com by clicking the blue 'sign in' button

    • Click 'Continue with browser'

      This will open the browser and present you a GitHub login screen - use the account details you set up previously

    • Once authenticated, click 'open GitHubDesktop' to navigate back to the GUI

  • Click 'Publish repository', verify the information presented (name and description of project you want to publish)

  • Click 'Publish repository' again, you will notice some activity happen

  • Verify, the publishing worked by visiting your GitHub account

  • You should notice the name of the repo appear in the left-hand sidebar

  • Open the repo in the browser to see if the two files created have been uploaded

    You have successfully connected the GUI to your GitHub account

  • Inside Windows File Explorer, copy and paste some files from a previous lab to demonstrate changes to the directory.

    As you are copying all the files over, this included the images and videos. In some instances its best practices not to upload media to GitHub - and to avoid GitHub processing these files we use the Git ignore file.

    Remember! Before you commit files to GitHub you need to perform 'flutter clean' in the terminal to remove any dependencies from your project else it will make the upload rather large

Activity 2: Integrating GitHub workflow in Visual Studio Code

  • Using Windows File Explore, create a new directory inside this weeks folder
  • Open Visual Studio Code
  • Open the directory you have just created (File > Open Folder)
  • Navigate to 'Extensions' and search for GitHub

  • Click install, the source control functionality should now appear in the left-hand sidebar (nodes with branching icon)

  • Click 'Initialize Repository'

    This is now initialising the directory you have opened in Visual Studio Code

  • Copy and paste the files from one of the lab to demonstrate changes to the directory - using Windows File Explorer

    You will notice in VSC that files have started to appear in the source control pane

    Remember! Before you commit files to GitHub you need to perform 'flutter clean' in the terminal to remove any dependencies from your project else it will make the upload rather large

  • Using the terminal (Terminal > New Terminal)

  • Type notepad.gitignore > (enter)

  • Follow any onscreen instructions to save the file

  • Click 'Always'

  • Provide a message - this should be something meaningful for instance, 'Learning how to integrate GitHub into my workflow for CO2509' > (enter)

    The files are now prepared ready to be published

  • Click 'Publish Changes'

    GitHub will need to authenticate the user. You should do this by clicking Allow.

  • This will open the web browser, click 'Allow'

  • Then login using your GitHub credentials created in lab 2

  • Once authenticated click 'Authorize github'

    You have now linked VSC with GitHub

  • Copy the token presented to you in case you need it

  • Open in Github Desktop > Open (allow an extension to open this URI)

  • VSC will prompt you for creating a public and private repository

    public means other people can see your code.

    private is where the repository is kept private and no one without access can view the files.

  • VSC may prompt you to supply the token (you copied) paste this into the connect to GitHub window > Sign in

  • Verify the files have been published to your GitHub account
