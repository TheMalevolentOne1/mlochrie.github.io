Introduction

GitHub has been around since 2008 and currently has 56 million users (to date). GitHub is built on Git (founded in 2005). Git is the protocol in which GitHub builds features around. GitHub is a hosting platform that helps software developers solve problems more efficiently, by providing tools for version control. One of the core benefits to adopting GitHub in your practice is when you are working in a team. GitHub makes collaboration simple and effective by allowing users to fork (taking a copy) code that then resides in their own repository (a user's collection of projects, aka a repo). GitHub also allows for users to make comments in peoples code or improvements to the code using (pull requests) notifying the main author of the code, which they may wish to update the main branch (a version of the instance). More importantly, GitHub keeps a record of everyone's contributions and feedback (version control). Version control is increasingly important as you progress through a a development of a project. This is a useful feature especially when considering the degree you are studying, where you make changes to a project over time, you might make many mistakes along the way (we all do), you have the confidence that you'll always have a backup of the most recent commit (the last time you published code). Commits are published source code that contains the code, a title and a description. The more context you provide the commit the better the more efficient the repo is.

Activity 3: Integrating GitHub Into your Workflow

So far you have learnt how to add a file manually through the GitHub website. GitHub allows much more than this, you can use plugins in your IDE to automatically commit as you save your work or through a Command Line Interface (CLI) or more recently GitHub have released their own Graphical User Interface (GUI) to monitor a folder on your computer and prepare the files ready for committing. It is expected you learn GitHub over the course of your study and integrate it into your learning. It is up to you to find the best method to integrate it into your learning.

There are plenty of guides to support you with your independent learning:

  • Git guides CLI
  • GitHub Desktop GUI
  • Working with GitHub in Visual Studio Code

This activity will demonstrate how to integrate GitHub commits using the GitHub Desktop application.

  • First of all sign into the desktop application using your GitHub credentials, by "clone a repository from the Internet..." > Sign in > Continue with browser, if you are logged into GitHub in your browser it should automatically relaunch and signed you into the desktop application. If you are not signed in, sign into GitHub.com and follow the on screen instructions

  • The application will load the repositories you are associated with. Find the one you've just created > select and "Clone [name of repo]"

  • You need to specify a local directory where you want the repo to be based. This is the location you will be developing the files to commit. Specify the local path and click "Clone"

  • Locate the local path ("Show in Explorer / Show in Finder")

    You will notice the README file you created for the assignment1 repo has now been fetched to your local machine

  • Open the README file in a text editor of your choice, make a simple update

  • Save the file

  • Navigate back to the GitHub Desktop application, you will see the change you've just made is reflected here and the "Commit to main" button is now active

  • Fill out the commit title and description

  • The commit is just one part of the process. You now need to push this commit to your repo. This is achieved by pressing the "Push origin" button

    Once it is complete the button is removed and you will notice the "Commit to main" button is now inactive

  • Check the change on GitHub.com to see if the commit and push worked

You have now hooked up a local directory to GitHub, any changes made to the local directly can be committed and pushed to GitHub. This includes adding any new files, updating code or creating sub folders within the folder structure.

To make this workflow simpler, you should investigate integrating GitHub into your IDE as it will make the process more efficient and effective when writing code. If you are struggling with this, Google the problem first then speak with your tutor.
