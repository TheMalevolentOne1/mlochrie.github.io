Activity 1. Button Component

  • Create a new Figma file

  • Using the shape tool → create a rounded rectangle

  • Now add the text "button" to the rectangle → centre it and select both items and group

  • Select the group and create a component (Ctrl + Alt + K)

    To access the quick action bar:

    • Mac: ⌘ Command / or ⌘ Command P

    • Windows: Control + / or Control + P

    Now you can search for the tool you require

  • Using the design sidebar and expand the properties menu → click variant (a purple highlight panel will form around the component)

  • Add a new variant by clicking on the '+' symbol (do this twice, so you have three instances of your button)

  • Style the buttons so that you have a default (active state), hover and inactive state

  • With all three buttons selected → change the name of the variant to "Primary" in the design sidebar by double-clicking on Property 1

    • Using the design sidebar rename the state of the variant → with the first variant selected to rename the state value to default

    • Repeat this for the second and third buttons (hover and inactive)

  • Create a frame and drag the button component to the view

  • Whilst selected in the prototype sidebar define the interaction for the button so that while hovering you change the state to hover

  • Test your component by hovering your mouse pointer over it

  • Create a set of button components for Secondary buttons using the same three states

Explore and discuss how you can extend componentisation using variants. You should be thinking about what components you might need for your assessment and build up your design system of these components with primary, secondary, and tertiary components. Components provide possibilities for scalable design systems - it is an industry best practice to understand how components can be used and re-used. For more information on preparing for variants see Figma's documentation.

Activity 2. Hiding Content

As you will have seen from the video, masks enable you to show areas of an object by 'masking' the other parts of the object that are selected. Another way to hide content is through the 'Clip Content' feature which allows you to hide objects that go beyond the frame bounds.

  • Using a new file, copy and paste an image into your artboard

  • Select a shape (Ellipse tool for example), draw the shape → place it over the image and arrange the shape behind the image (you can do this by moving the position of the layer in the layers panel or right-clicking and 'send to back')

  • Select both the image and shape and click the mask icon on the toolbar (two circles overlapping)

  • You should now have an image mask. Customise the mask by adding more smaller shapes on top of the image (like bubbles floating around your image)

  • For best practice ensure you have grouped your items and named the layers

Activity 3. Sticky Elements

You might have heard about sticky headers or footers from web design. This is where the head/bottom navigation sticks to the top of the view when scrolling. This is a popular design philosophy in mobile application development. In order to have sticky elements you need to create scrolling content.

  • Create a new Figma file → Using the frame tool, create a frame for a mobile device

  • Add another frame inside the artboard (name the layer 'text container') → select and drag a text label and fill this with some Lorem Ipsum and rename the layer 'text' → make sure the text and container fill the view height and width

  • On the text container layer ensure that 'Clip Content' is selected → ensure that you have enough text so that it overflows outside of your frame container. Notice what happens to the content when this is selected

  • With the text container layer selected, click on the prototype tab go to Overflow Behaviour and select vertical scrolling

  • Play the prototype and test to see if the content is scrollable

Fix position when scrolling: to create sticky elements you need to lock the position of the element to make sure the object's position remains the same.

  • Add a rectangle shape to fill the bottom of the view (acting like a navbar) set the background colour to #FFFFFF, and name this layer navbar

  • Set the constraints of this to bottom, left → select 'fix position when scrolling'

  • This should now appear above your text layer, decrease the opacity to the navbar to 50% and add a background blur of 10

  • Test the prototype

    You will notice that there is part of the text that stays behind the navbar and you can't scroll to it

  • This is because you have clip content selected. If you untick this for the text container and re-run your prototype you should now be able to scroll all the way to the end of the content

See Figma's documentation on scrolling and overflow
